## 层 & 神经元 & 参数/权重 & 激活值
    + 层
        + 在 Transformer 架构中，“层”通常指一个解码器块（Decoder Block）。
        + 它内部包含两个核心子模块：自注意力机制（Self-Attention）和前馈网络（FFN/MLP）。
        + 为什么要分层？
            + 深度的“层”堆叠是目前实现涌现能力（Emergent Abilities）的最优解。
            + 单层网络只能处理简单的线性或浅层逻辑，无法理解人类语言中复杂的语境和因果。即使有 270 亿个参数，如果全部堆在一个层里（即“极浅但极宽”的模型），它只能记住表面模式（死记硬背），而无法产生推理能力。缺乏分级抽象能力，无法模拟大脑处理信息的阶梯式逻辑。
            + 底层逻辑：数据向量进入第 1 层，进行初步加工；输出结果作为第 2 层的输入，进一步精炼。这就像写文章：第一遍打草稿，第二遍修辞，第三遍润色。每一层都在原有的基础上叠加更复杂的理解。
            + 极多但极窄的层（如 1000 层，每层参数极少）呢？计算效率极低（必须串行），且容易出现梯度消失问题，导致模型难以训练。
    + 神经元（Neuron）：
        + 在每一层包含数千个神经元，而每个神经元又由成千上万个参数（权重）定义。
        + 一个神经元是一根管道，而参数是控制这根管道进水量的“阀门旋钮”。一个神经元通常连接着许多个权重参数（由上一层输入的每一个维度与其相连）。
        + 每一个维度（Dimension）就代表一个神经元，也对应计算张量中的一个通道（Channel）。
    + 27B：代表模型全身上下所有层加起来共有约 270 亿个数字（权重参数）。这些参数分布在几十个连续排布的 Decoder Blocks 中。
    + 激活值：
        + 是模型在推理（Inference）过程中，输入数据与每一层权重矩阵相乘（并经过激活函数）后生成的中间状态张量。
        + 是整层参数共同作用于输入数据后，产生的一组新数据，准备传给下一层。对应的是层与层之间的中间输出。
        + 激活值的大小取决于输入序列的长度（Sequence Length）。如果输入1个token，每一层输出的激活张量大小通常是 [1, 隐藏层维度]。如果你输入1024个token，激活张量就是 [1024, 隐藏层维度]。
        + 如果有 20 个 Transformer Block，数据会依次流过这 20 个站。在每一层之后，都会产生一个完整的激活张量。如果模型隐藏层维度是 5120，输入了 10 个 Token，那么每一层产生的激活值是一个 $10 \times 5120$ 的矩阵。输入 $\rightarrow$ [Layer 1] $\rightarrow$ 激活值 1 $\rightarrow$ [Layer 2] $\rightarrow$ 激活值 2 $\dots$
    

## 传播

推理（Inference）：仅执行前向传播。  
训练（Training）/微调（Fine-tuning）：包含前向传播、计算损失、反向传播及参数更新。

+ 损失计算（Loss Calculation）—— “对答案”：
    + 对象：预测值、真实标签（Label）。
    + 本质：计算模型错得有多离谱。
    + 将模型前向传播输出的预测值（Prediction）与数据集中自带的真实标签（Label/Ground Truth）进行数学对比，计算出一个标量数值（Loss）。

+ 反向传播与参数更新
    + 计算得到损失后，反向传播则能得到 27B 参数中，每一个参数该为这个错误负多少责任。
    + 利用微积分的链式法则，将损失值从输出层逆向传回输入层，计算每一层参数的梯度，并将梯度作为“前进的方向”。它告诉参数：“如果你往这个方向动一点，损失就会减小”，以此来更新参数。
    + 先算最后一层的梯度，然后利用它算倒数第二层，以此类推，一次性走完所有层，不是走一层计算梯度并更新参数。因此参数更新不是“逐层实时进行”的。在一次反向传播中，会先算出所有层的梯度，形成一个完整的“梯度集合”，然后统一对所有参数进行一次微调。

### 训练（Training）/微调（Fine-tuning）闭环过程
1. 前向传播（Forward）—— “答题”：
    + 对象：输入数据、权重参数、激活值。
    + 过程：数据一层层传递，每一层根据当前的权重对数据进行变换。
    + 结果：产生一个预测值。
2. 损失计算（Loss Calculation）—— “对答案”：
    + 对象：预测值、真实标签（Label）。
    + 本质：计算模型错得有多离谱。
3. 反向传播（Backward）与梯度 —— “改错”：
    梯度（Gradient）的通俗解释：想象你在大雾弥漫的山上（误差高地），你想下山到谷底（误差最小点）。你看不见远方，只能用脚试探周围 360 度的坡度。梯度就是那个“最陡峭的上升方向”。为了下山，你要向梯度的反方向迈步。
    梯度大：说明这一步改动对结果影响巨大。
    梯度小：说明已经接近最优，或者这个参数不太重要。
    + 微调（Fine-tuning）：本质上就是反向传播。它是指在一个已经学会很多知识的模型基础上，再次运行反向传播，让它针对特定任务（比如写代码）微调它的梯度，从而更适应新场景。


### 前向传播中的信息演变

在前向传播中，模型每经过一个隐藏层（Hidden Layer），并不会生成一个“词”。相反，它会产生并不断更新一个名为激活值（Activation）的隐藏向量。直到数据走完所有预设的层，并经过最后的输出头（LM Head）和采样算法，才会最终确定一个“词”。

+ 激活值 (Activation) ⚡	神经元在处理特定输入时产生的中间计算结果。量化算法观察的对象。通过观察 512 个样本产生的激活值，确定量化的缩放比例（Scale）。在模型前向传播过程中，每一层神经网络（层）接收输入数据，经过矩阵运算和激活函数后，输出给下一层的数据，即为激活值。激活值记录了模型在“思考”过程中动态产生的数值波动。权重（Weight）是静态的“知识”，而激活值是动态的“逻辑流”。



### 反向传播


+ 反向传播：从输出层开始，利用微积分中的链式法则，逆向计算每一层参数对最终误差（损失函数）的贡献度。
+ 为什么要进行反向传播
    + 前向传播告诉我们模型“错了多少”（误差），但它没法告诉是 27B 参数里的哪一个数字导致了错误。
    + 链式法则 (Chain Rule) 🔗：这是“传声筒”。如果我们知道 A 对 B 的影响，B 对 C 的影响，我们就能算出 A 对 C 的影响。反向传播就是把输出端的误差，通过一层层的导数乘积，精准地传回输入端。

+ 梯度 (Gradient)：
    + 是一个向量，指向函数值增长最快的方向。在训练中，我们计算它，然后向它的反方向迈出一小步（梯度下降）。
    + 本质属性：它是损失函数（损失函数 (Loss Function) 📉：它是“裁判”。它定义了预测值与真实值之间的差距，是所有计算的起点。）关于参数的导数集合
    + 在深度学习工程中，我们只在反向传播阶段显式地计算并利用它。

+ 梯度消失是指在神经网络的反向传播（Backpropagation）过程中，计算得到的梯度（用于更新参数的信号）随着层数增加而变得越来越小，最终接近于零，导致靠近输入层的参数几乎无法得到更新。


## 架构
FFN（前馈网络）
    + Dense FFN。所有输入必须经过同一个复杂的矩阵运算。

MoE（混合专家）
    + 1 个 Router 和 $N$ 个 Experts。将一个巨大的 FFN 拆分成了多个小的 FFN（即“专家”），并增加了一个路由（Router）。
    + 串联逻辑：输入 $\rightarrow$ Router 计算权重 $\rightarrow$ 选择最匹配的 1~2 个 Experts（FFN）进行计算 $\rightarrow$ 结果加权求和。
    + 传统的 Dense 模型（如 Llama 3-8B）每一层只有一个固定的 FFN 块。而 MoE 的核心价值在于“稀疏性”
