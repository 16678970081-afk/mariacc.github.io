## 校准机制

### 什么是校准

校准是指将一小部分代表性数据输入模型，通过查看激活分布，进而计算出此次量化的缩放因子。
+ 具体机制：记录各层权重和激活值的分布范围（Min/Max/标准差），并据此计算出量化缩放因子（Scale Factor，让量化前的哪个数对应量化后的哪个数）的过程。
+ 为什么需要计算缩放因子：因为量化的本质是“映射”，把分布在 $(- \infty, + \infty)$ 的浮点数映射到有限的整数区间（如 INT8 的 $[-128, 127]$）。
+ 校准解决的是“怎么映射最不失真”的问题。如果不做校准，强制截断数值，模型会立刻出现严重的精度衰减（“智力倒退”）。
校准的完整机制实际上是一个“求取最大公约数”的过程，力求让有限的位宽尽可能完整地保留原始数据的统计特征。

+ 激活并非“越多越好”，而是“越具代表性越好”。
    + 激活参数空间的意思：如果使用随机生成的乱码，理论上可以激活所有参数，但这叫“噪声”，它产生的激活分布是杂乱无章的。
    + AWQ 的核心假设：权重本身并不都同等重要，激活值大的权重才重要。
    + 缩放因子的准确性：
        + 如果数据能覆盖你未来实际使用场景的参数空间，那么缩放因子就是“最准确”的。
        + 如果激活了太多无关的参数空间（比如用小说数据校准航天模型），AWQ 可能会把宝贵的精度指标浪费在“描写修辞”的权重上，反而挤占了“物理计算”权重的精度。

### 量化中的激活值
+ 激活值在量化中的特殊作用：
    + 在 AWQ 量化中，激活值被赋予了“探测器”的使命。通过观察它，能发现哪些权重在实际运算中“压力最大”（数值最极端），从而决定如何分配有限的 4-bit 精度空间。如果不参考激活值进行量化，模型就会像盲人摸象，极易把关键的微小信号当成噪声抹除。

    + 感知激活（AWQ）
        通过真实数据流（校准集）产生的“水压”（激活值），反向定位核心权重，精度保持能力高。

+ 激活值的生命周期
    + 在标准推理（Inference）中：它遵循硬件效率优先的“机械逻辑”。用完即焚。为了极速响应，显存只保留“当前处理层”和“KV Cache（用于生成后续 Token）”。激活值像接力棒，传给下一层后，上一层的使命就结束了。
    + 在量化校准或反向传播（Training/Calibration）中：
        + AWQ 量化校准（局部保留）
        + 底层机制：AWQ 算法需要计算每个通道的激活分布。因此，在校准阶段，程序会暂时“拦住”这些激活值，统计它们的绝对值最大值等指标，存入一个专门的统计表（通常很小），然后才释放原始的巨大张量。
        + 输入数据 $\rightarrow$ 产生高维激活值 $\rightarrow$ AWQ 统计器捕捉分布特征 $\rightarrow$ 丢弃激活值 $\rightarrow$ 进入下一层。
    + 在标准的 Transformer 前向传播中，激活值通常是流式传递的。当第 $L$ 层的输出（激活值）计算完毕并传递给第 $L+1$ 层作为输入后，如果没有后续任务（如梯度计算）需要这些数据，第 $L$ 层的激活值所占用的内存空间会被标记为可释放或直接被下一层的数据覆盖。



### 校准数据的选择

校准数据集的选择通过影响缩放因子，进而影响量化后模型在某一特定领域的回答的准确度

以航天领域为例：
1. 输入数据：选择航天领域的校准数据。
2. 触发激活：模型处理这些数据时，与航天物理、轨道计算相关的神经元（管道）会产生极高的激活值（高水压）。
3. 确定缩放因子（Scaling Factor）：AWQ 监测到这些高水压点，会意识到：“这几个管道非常重要，不能轻易压缩！” 于是它会计算出一组特定的缩放因子，给这些显著权重（Salient Weights）预留更多的精度空间。
4. 量化执行：在 4-bit 的有限格子中，航天相关的权重得到了更好的保留。
5. 结果：量化后的模型在回答“卫星如何变轨”时，精度损失极小；但在回答“如何做红烧肉”时，由于相关神经元在校准时没被充分激活，精度损失可能会大一些。

校准数据集在哪一部分起到了关键作用？ 
+ 核心机制：在 寻找 Scale（缩放比例） 这一步。
+ 串联逻辑：
    1. 输入校准集：让模型跑一遍 Forward（前向传播）。
    2. 观察激活：记录每个层、每个通道的激活值最大值 $X_{max}$。
    3. 计算保护位：AWQ 会根据 $X_{max}$ 识别出哪些权重是“显著权重”（Salient Weights）。
    4. 权重重缩放：给这些显著权重更大的“生存空间”，从而在压成 4-bit 时减少信息丢失。
    结论：数据集是观察激活分布的“窗口”。

什么是“激活参数空间”？
+ 参数空间：指模型中所有可以学习的权重（Weights）。比如 Qwen2.5-27B 有 270 亿个参数。
+ 激活：指让这些平时“静止”的参数在推理时产生输出。
+ 全意理解：
    大模型就像一个巨大的图书馆。如果你只问它“你好”，它可能只动用了入口处的几个书架（少量参数激活）。但 ultrachat  包含复杂指令（如：请帮我写一个复杂的 Python 脚本并解释逻辑）和多轮对话，这迫使模型调动大脑深处的知识储备、逻辑推演和语言组织能力。

底层逻辑：复杂的任务会产生更丰富、更极端的激活值。只有把模型的各层、各个通道都“跑一遍”，AWQ 才能识别出哪些权重在处理复杂任务时是不可或缺的（即“显著权重”），进而给予特殊保护。

航天 VLM 模型定制一个校准集

AWQ为什么需要校准：
AWQ 属于训练后量化 (PTQ)。量化过程会产生误差，而 AWQ 的精髓在于它不是盲目量化，而是通过一小部分真实数据（校准集）跑一遍前向传播，观察模型内部神经元的激活值 (Activation) 分布。
如果不做： 量化器将无法识别哪些权重是“显著”的（即对应大激活值的权重），从而导致量化误差均匀分布，最终令模型输出乱码或逻辑崩溃。


## 专家模式 MoE

+ 专家模块 (Expert) ： MoE 架构中负责特定知识领域的“专业小组”。	qwen3.5 - 27B 模型中有多个专家，我们需要足够的样本确保每一个专家都被激活过。
+ 门控 (Gating/Router) ：模型内部的“交警”，决定将当前 Token 发给哪个专家。	决定了样本数据在模型内部的流向。
