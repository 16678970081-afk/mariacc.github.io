## **Docker**

### **安装docker-阿里云镜像**

**win11+wsl2:**

1. wins下安装docker desktop (通过docker启动dify）
    1. docker官网下载amd64版本[https://www.docker.com/](https://www.docker.com/)
    2. 配国内镜像源
    3. 集成wsl：确认settings--resources--wsl integration已勾选

**Ubuntu 24:**

1：安装工具

sudo apt install -y ca-certificates curl gnupg

2：添加阿里云 Docker 镜像源的签名密钥

sudo install -m 0755 -d /etc/apt/keyrings

curl -fsSL [https://mirrors.aliyun.com/docker-ce/linux/ubuntu/gpg](https://mirrors.aliyun.com/docker-ce/linux/ubuntu/gpg) | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

sudo chmod a+r /etc/apt/keyrings/docker.gpg

3：添加阿里云 Docker 软件源

```
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://mirrors.aliyun.com/docker-ce/linux/ubuntu \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
```

4：刷新软件列表并安装 Docker

sudo apt update

sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

5：**设置镜像挂载到数据盘、配置镜像源**

确定数据盘位置：df -h （挂载点）

停止服务：sudo systemctl stop docker

sudo systemctl stop docker.socket

编辑/创建 守护进程配置文件

sudo mkdir -p /etc/docker

`sudo nano /etc/docker/daemon.json`：

```bash
{
  # 镜像存储地址
  "data-root": "/data/docker",
  # 镜像源
  "registry-mirrors": [
    "https://mirror.ccs.tencentyun.com",
    "https://docker.mirrors.ustc.edu.cn",
    "https://mirrors.tuna.tsinghua.edu.cn/",
  ],
  "insecure-registries": [
  ],
  "debug": true,
  "experimental": false
}

# 重启
sudo systemctl daemon-reload
sudo systemctl restart docker
```

- 阿里云镜像加速器注册：容器镜像服务--镜像工具--镜像加速器--复制地址
- 不要用陌生镜像源

6: 把账户加入 docker 用户组,之后每次运行docker命令前面不需要加sudo

sudo usermod -aG docker hello

6：设置 Docker 开机自启

sudo systemctl enable --now docker

7: 重启重新登陆：刷新用户组

8：验证 Docker 安装成功

输出：Docker version 27.x.x

### **安装**NVIDIA Container Toolkit

- Docker 默认隔离了硬件，装了这个工具，容器才能调用显卡。

```bash
# 添加 NVIDIA 容器工具包的软件源（告诉系统去哪里下载）
curl -fsSL https://nvidia.github.io/libnvidia-container/gpgkey \
  | sudo gpg --dearmor -o /usr/share/keyrings/nvidia-ct-keyring.gpg

curl -s -L https://nvidia.github.io/libnvidia-container/stable/deb/nvidia-container-toolkit.list \
  | sed 's#deb https://#deb [signed-by=/usr/share/keyrings/nvidia-ct-keyring.gpg] https://#g' \
  | sudo tee /etc/apt/sources.list.d/nvidia-container-toolkit.list

# 安装工具包
sudo apt update && sudo apt install -y nvidia-container-toolkit

# 配置 Docker 使用 NVIDIA 运行时
sudo nvidia-ctk runtime configure --runtime=docker

# 重启 Docker 让配置生效
sudo systemctl restart docker
```

验证

```bash
# 在 Docker 容器里运行 nvidia-smi，如果看到显卡信息说明一切正常
docker run --rm --gpus all nvidia/cuda:12.8.0-base-ubuntu24.04 nvidia-smi
```

### 相关命令

1. 查看docker 根目录：docker info | grep -i "docker root dir"

| 子目录         | 存什么            |
| ----------- | -------------- |
| image/      | 镜像的元数据（索引信息）   |
| overlay2/   | 镜像和容器的实际文件层    |
| containers/ | 每个运行过的容器的配置和日志 |
| volumes/    | Docker 数据卷     |

2. docker引擎，镜像文件挂载点：df -h 【docker info | grep -i "docker root dir"输出结果】
3. docker总体磁盘占用：docker system df
4. docker引擎等文件从系统盘到数据盘迁移：
    1. 停止 Docker 服务

```bash
cd ~/ai-stack
sudo docker compose down

sudo systemctl stop docker
sudo systemctl stop docker.socket
# 在数据盘上创建新目录
sudo mkdir -p /data/docker
#迁移
sudo rsync -aP /var/lib/docker/ /data/docker/
#添加docker配置/etc/docker/daemon.json
{
  "data-root": "/data/docker"
}
sudo systemctl start docker
#验证
docker info | grep "Docker Root Dir"
docker images
```

5. 查看所有容器挂载点：`docker inspect $(docker ps -q) --format '{{.Name}}: {{range .Mounts}}{{.Source}}->{{.Destination}} {{end}}`
6. 看挂载点占用空间大小：sudo du -sh 【上一步的地址】

./ollama-modelscope-install.sh

4. 运行ollama

ollama serve &

5. 验证ollama正常运行

curl [http://localhost:11434](http://localhost:11434) ("Ollama is running.")

### 通过Docker安装（国内镜像源）

1. 配置docker国内镜像源
2. docker-compose

```bash
services:
  # ===== Ollama：AI 模型推理引擎 =====
  ollama:
    image: ollama/ollama:latest        # 使用官方最新镜像
    container_name: ollama
    runtime: nvidia                    # 使用 NVIDIA GPU 运行时
    environment:
      - NVIDIA_VISIBLE_DEVICES=all     # 允许使用所有 GPU
      - OLLAMA_KEEP_ALIVE=-1           # 模型永不从显存卸载（保持响应速度）
      - OLLAMA_HOST=0.0.0.0            # 在容器内监听所有网络接口
    volumes:
      - /data/models:/root/.ollama     # 模型文件存到 8TB 硬盘
      # 模型等数据文件挂载地址：/data/models
    ports:
      - "127.0.0.1:11434:11434"        # 仅本机可访问，不对外暴露
    restart: unless-stopped            # 除非手动停止，否则崩溃后自动重启

```

3. docker compose pull ollama
4. docker compose up -d ollama

## 其他

### gitee加速github下载

1. 复制github项目链接
2. 切换到 Gitee，在右上角点击「+」号，选择「从 GitHub / GitLab 导入仓库」。
3. 在导入界面中，将 GitHub 的链接贴入「Git 仓库 URL」中。这时候有可能会出现提示，例如：

> 检测到目前导入仓库在 Gitee 上已有公开同步仓库。

如果看到这样的提示，说明之前已经有人导入过了，这时可以直接使用之前用户导入的链接，替换掉 GitHub 的链接进行克隆。如果担心别的用户导入的内容会在未来被删掉，可以手动选择导入到自己的账户下。

3. 如果选择导入到自己的账户，需要在下方的是否开源选项中勾选「公开」，然后点击导入。
4. 导入完成后，直接使用 Gitee 的链接替换掉 GitHub 原先的链接进行克隆，进行加速。
