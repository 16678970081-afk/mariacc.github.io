## 网络配置

通过nginx反向代理：

```bash
# 安装nginx
sudo apt install -y nginx
nginx -v
# 确认本机内网 IP
ip addr show | grep 'inet ' | grep -v '127.0.0.1'
# 生成自签名 TLS 证书（有效期 10 年）
# 1. 创建证书目录
sudo mkdir -p /etc/nginx/ssl

# 2. 生成自签名证书
sudo openssl req -x509 -nodes -days 3650 -newkey rsa:4096 \
  -keyout /etc/nginx/ssl/ai.key \
  -out /etc/nginx/ssl/ai.crt \
  -subj "/C=CN/ST=Beijing/O=Company/CN=10.139.105.12" \
  -addext "subjectAltName=IP:10.139.105.12,IP:127.0.0.1"
  
# 创建nginx配置文件
sudo nano /etc/nginx/sites-available/ai-stack
```

- server_name要换成本机内网IP
- openclaw网页端添加路由接口

```
# HTTP → HTTPS 跳转
server {
    listen 80;
    server_name 【本机内网IP】;
    return 301 https://$host$request_uri;
}

# HTTPS 主配置
server {
    listen 443 ssl;
    server_name 【本机内网IP】;

    # TLS 证书
    ssl_certificate     /etc/nginx/ssl/ai.crt;
    ssl_certificate_key /etc/nginx/ssl/ai.key;
    ssl_protocols       TLSv1.2 TLSv1.3;

    # 只允许内网 IP 段访问，拒绝一切外部访问
    allow 192.168.0.0/16;   # 内网 C 类网段
    allow 10.0.0.0/8;       # 内网 A 类网段
    allow 172.16.0.0/12;
    allow 127.0.0.1;
    deny  all;              # 其余全部拒绝

    # / 路由 → Open WebUI 聊天界面
    location / {
        proxy_pass         http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header   Upgrade $http_upgrade;
        proxy_set_header   Connection 'upgrade';
        proxy_set_header   Host $host;
        proxy_set_header   X-Real-IP $remote_addr;
        proxy_set_header   X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header   X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        # AI 流式回答需要关闭缓冲，否则会攒一大段才一次性返回
        proxy_buffering    off;
        # 大模型生成长回复可能耗时较久，10 分钟超时兜底
        proxy_read_timeout 600s;
        proxy_send_timeout 600s;

    }

    # /claw/ 路由 → OpeClaw Agent 接口
    location /claw/ {
        proxy_pass         http://127.0.0.1:18789/;
        proxy_http_version 1.1;
        proxy_set_header   Host $host;
    }
}


# WebSocket 升级映射

map $http_upgrade $connection_upgrade {
default upgrade;
''      close;
}
```

- listen 80 自动 301 跳转 HTTPS
- listen 443 ssl 启用 TLS 加密
- allow 192.168.0.0/16 + 10.0.0.0/8 只放行内网 IP
- deny all 拒绝一切非内网连接
- location / proxy_pass 到 `http://127.0.0.1:3000`（Open WebUI）

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/ai-stack /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/default
sudo nginx -t
sudo systemctl reload nginx
sudo systemctl enable nginx
```

- 本机验证：`curl -k https://127.0.0.1/`，应返回 Open WebUI HTML

```bash
services:
  # ===== Ollama：AI 模型推理引擎 =====
  ollama:
    image: ollama/ollama:latest        # 使用官方最新镜像
    container_name: ollama
    runtime: nvidia                    # 使用 NVIDIA GPU 运行时
    environment:
      - NVIDIA_VISIBLE_DEVICES=all     # 允许使用所有 GPU
      - OLLAMA_KEEP_ALIVE=-1           # 模型永不从显存卸载（保持响应速度）

    volumes:
```

    1. 查看WSL 发行版名称（通过 `wsl -l -v` 查看）：Ubuntu-24.04
    2. wsl -d 【WSL 发行版名称】 -- bash -c "cd ~/dify/docker && docker compose up -d"
3. 在 Windows 上创建脚本文件： `C:\Scripts\wsl2-port-forward.ps1`

```bash
$wslIp = (wsl -d [WSL发行版名称] -- hostname -I).Trim().Split(" ")[0]

if (-not $wslIp) {
    Write-Host "无法获取 WSL2 IP，退出"
    exit 1
}

Write-Host "WSL2 IP: $wslIp"

# 删除旧的端口转发规则
netsh interface portproxy delete v4tov4 listenaddress=0.0.0.0 listenport=80

# 添加新的端口转发规则（指向最新的 WSL2 IP）
netsh interface portproxy add v4tov4 listenaddress=0.0.0.0 listenport=80 connectaddress=$wslIp connectport=80

```

- 验证：
    - 查看转发规则：netsh interface portproxy show all
    - 查看WSL地址：wsl -d 【WSL 发行版名称】 -- hostname -I
    - 二者相同

    - 配置为：Windows 10/11
4. 触发器选项卡：
    - 新建 -> 选择"登录时"
    - 可选：延迟任务 30 秒（给 WSL 和 Docker 足够启动时间）
5. 操作选项卡：
    - 新建 -> 启动程序
    - 程序输入：`powershell.exe`
    - 参数输入：`-ExecutionPolicy Bypass -File "C:\Scripts\wsl2-port-forward.ps1"`
6. 条件选项卡：
    - 取消勾选"只有在计算机使用交流电源时才启动"
7. 确认保存

```powershell
New-NetFirewallRule -DisplayName "WSL2 Dify" -Direction Inbound -LocalPort 80 -Protocol TCP -Action Allow
```
