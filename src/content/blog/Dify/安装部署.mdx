d: 其他从同一局域网内访问应用（永久）

1. docker desktop设置开机启动
2. 设置dify docker compose开机自启动（win：）
    1. 查看WSL 发行版名称（通过 `wsl -l -v` 查看）：Ubuntu-24.04
    2. wsl -d 【WSL 发行版名称】 -- bash -c "cd ~/dify/docker && docker compose up -d"
3. 在 Windows 上创建脚本文件： `C:\Scripts\wsl2-port-forward.ps1`

```bash
$wslIp = (wsl -d [WSL发行版名称] -- hostname -I).Trim().Split(" ")[0]

if (-not $wslIp) {
    Write-Host "无法获取 WSL2 IP，退出"
    exit 1
}

Write-Host "WSL2 IP: $wslIp"

# 删除旧的端口转发规则
netsh interface portproxy delete v4tov4 listenaddress=0.0.0.0 listenport=80

# 添加新的端口转发规则（指向最新的 WSL2 IP）
netsh interface portproxy add v4tov4 listenaddress=0.0.0.0 listenport=80 connectaddress=$wslIp connectport=80

```

- 验证：
    - 查看转发规则：netsh interface portproxy show all
    - 查看WSL地址：wsl -d 【WSL 发行版名称】 -- hostname -I
    - 二者相同

d: 设置 Windows 开机自动执行该脚本

1. 打开"任务计划程序" (`taskschd.msc`)
2. 点击"创建任务"（不是"创建基本任务"）
3. 常规选项卡：
    - 名称：`WSL2 Port Forward`
    - 勾选"使用最高权限运行"（必须，因为 netsh 需要管理员权限）
    - 配置为：Windows 10/11
4. 触发器选项卡：
    - 新建 -> 选择"登录时"
    - 可选：延迟任务 30 秒（给 WSL 和 Docker 足够启动时间）
5. 操作选项卡：
    - 新建 -> 启动程序
    - 程序输入：`powershell.exe`
    - 参数输入：`-ExecutionPolicy Bypass -File "C:\Scripts\wsl2-port-forward.ps1"`
6. 条件选项卡：
    - 取消勾选"只有在计算机使用交流电源时才启动"
7. 确认保存

e. 确保 Windows 防火墙放行 80 端口

```powershell
New-NetFirewallRule -DisplayName "WSL2 Dify" -Direction Inbound -LocalPort 80 -Protocol TCP -Action Allow
```

- 运行后重启生效
